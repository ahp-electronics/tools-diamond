USERCODE

Specifies a code for storing device data such as firmware version number, manufacturerâ€™s ID, programming date, programmer make, pattern code, or JEDEC file checksum. USERCODE assists with record maintenance and product flow control. The USERCODE format can be binary, hexadecimal, or ASCII. For LatticeXP/XP2 and MachXO devices, CHECKSUM can be used as USERCODE. For ECP5 devices, the Auto option is available, which enables you to specify a UNIQUE_ID for the design.

Device Support

All

Syntax

USERCODE <usercode_format> <usercode>;

USERCODE CHECKSUM;

USERCODE AUTO;

where:

<usercode_format> ::= BIN | HEX | ASCII | AUTO

Specifies one of the available formats for codes: binary (BIN), hexadecimal (HEX), text (ASCII), AUTO

<usercode> ::=  string

Observe the following guidelines when specifying strings for user codes:

For binary or BIN format, specify a 32-bit user code string using only 1 or 0 digits.
For hexadecimal or HEX format, specify an eight-character user code string using only 0 through F values. If you enter a lowercase letter value, it will be converted to uppercase.
For text or ASCII format, specify a four character user code string using only alpha and numeric values. Characters are case sensitive.
For LatticeXP/XP2 and MachXO devices, CHECKSUM can be specified as user code in the logical preference file or in the Global Preferences sheet of Spreadsheet View.
Note
If you specify fewer characters than expected, the software will pad the code with zeroes to the correct number of characters. The software will not accept codes that exceed the expected number of characters for a given format.

In addition to the physical preference file (.prf), USERCODE is recorded in the data file: the JEDEC file (.jed) for non-volatile devices or the bitstream file (.bit) for volatile devices. The USERCODE can be read back by means of the JTAG port using the Diamond Programmer, a third-party programmer, or an embedded system.

Examples

The following are examples of USERCODE preference usages for the various formats.

USERCODE BIN "01010011011001010110110101101001";
USERCODE HEX "53656D69";
USERCODE ASCII "Semi";
USERCODE CHECKSUM;
USERCODE AUTO;

Each of these USERCODE preference examples is reflected in the data file. For binary format (BIN), the JEDEC file will show U00000000000000000000010000011111* [ bit order[31:0]. Notice that the USERCODE is padded by zeros on the left. For HEX format, the JEDEC file will contain the code UHEF456ADFF*. For ASCII format, the JEDEC file will contain the code UAanytext*. CHECKSUM will appear as HEX format in the JEDEC file.

For bitstream files, the USERCODE will only be stored in binary, since a bitstream is a binary file.

Refer to the Programmer online Help for more information about the use of USERCODE, CHECKSUM, and data files.
