RAM_DQ

Description:

Implements a RAM using EBR (Embedded Block RAM). Can support registered and unregistered outputs.

Devices Supported:

ECP5U, ECP5UM, LatticeEC, LatticeECP, LatticeECP2, LatticeECP2M, LatticeECP3, LatticeSC, LatticeSCM, LatticeXP, LatticeXP2, LIFMD, LIFMDF, MachXO, MachXO2, MachXO3D, MachXO3L, Platform Manager 2

Version:

7.6

Revision History:

7.6: Added LIFMDF support.

7.5: Added LIFMD support. Added MachXO3D support.

7.4: Added ECP5 support.

7.3: Added MachXO3L support.

7.2: Added support for comments and empty lines in memory initialization files.

7.1: Set unused inputs to GND to aviod unnecessary setup checks.

7.0: Added Area/Speed option and modified attributes for enhanced post-par memory re-initialization support. Added pipeline option for ECC outputs. Removed GSR option.

6.0: Removed READ_BEFORE_WRITE (all devices except LatticeEC, LatticeECP, and LatticeXP).

5.0: Added Byte enable. Added "wire" declaration in Verilog output.

4.1: Changed default reset mode to be SYNCH.

4.0: Added ECC support. Additional device support.

3.1: Comment added to .v and .vhd output files giving a link to a Technical Note on the Lattice Semiconductor Web site.

3.0: Added attributes in HDL that are required for post-PAR memory initialization.

2.0: Updated for compatibility with IPexpress.

1.0: Original released version.
