DDR Generic

Description:

Allows you to generate a DDR for a generic application.

Devices Supported:

ECP5U, ECP5UM, LatticeEC, LatticeECP, LatticeECP2, LatticeECP2M, LatticeECP3, LatticeXP, LatticeXP2, LIFMD, LIFMDF, MachXO2, MachXO3D, MachXO3L, Platform Manager 2

Version:

6.1

Revision History:

6.1: Added LIFMDF support.

6.0: Switched primitive lib to synthesis header. Added LIFMD support. Added MachXO3D support.

5.8: Added support for IOL on ECP5 with uneven sides.

5.7: Added MachXO3L MIPI support. ECP5 update.

5.6: Updated synchronization logic. Added ECP5 support.

5.5: Added MachXO3L support.

The DEL_ADJ and DEL_VAL attributes can no longer be accessed in the HDL code.

5.4: Bug fixes and minor updates.

5.3: LatticeECP4 update.

5.2: Minor updates.

5.1: Added GDDRX1_RX.SCLK.PLL.Aligned and GDDRX2_RX.ECLK.Aligned (no CLKDIV) for LatticeECP3.

5.0: Major changes for LatticeECP3.

4.0: For LatticeECP3, removed bypass delay mode, output 2x, and all bidi modes.

3.6: For LatticeECP3, removed ECLK from DQSBUFF and DQSBUFG.

3.5: For LatticeECP2/M and LatticeXP2, fixed the port-order for input DDRs using a gearing ratio of 2. Updated LatticeECP3 elements.

3.4: Added "wire" declaration in Verilog output.

3.3: Added support for CLKDIV (ECP2); Added syn_noprune for I/O Assistant flow (ECP2).

3.2: For LatticeECP2 and LatticeXP2, directional DDRs input and output registers use CE='1' and RST='0'.

3.1: Additional device support; single clock option for x1 gear Inputs and Bidi.

3.0: For LatticeECP2: Fixed delay mode allows user to specify the delay. Enable and reset are changed to meet hardware requirements. For LatticeECP/EC, and LatticeXP, there is no change.

2.0: Updated for compatibility with IPexpress.

1.0: Original released version.
